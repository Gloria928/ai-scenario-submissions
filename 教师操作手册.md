## 🚀 系统概览


## 📊 教师管理功能

### **1. 查看系统状态**
访问健康检查接口：
```
https://ai-scenario-submissions.onrender.com/health
```

**返回信息包括**：
- 系统运行状态
- 提交数量统计
- 学生活跃度
- 系统运行时间

### **2. 查看课堂统计**
访问统计接口：
```
https://ai-scenario-submissions.onrender.com/api/stats
```

**统计信息包括**：
- 总提交数量
- 活跃学生数
- 按作业分类统计
- 按编程语言统计
- 最近10次提交记录

### **3. 管理GitHub作业**
访问GitHub仓库 Issues：
```
https://github.com/Gloria928/ai-scenario-submissions/issues
```

**可以**：
- 查看所有学生提交
- 查看自动评分结果
- 手动添加评语
- 关闭已完成的作业

### **4. 监控自动评分**
访问GitHub Actions：
```
https://github.com/Gloria928/ai-scenario-submissions/actions
```

**监控**：
- 评分工作流运行状态
- 评分成功/失败情况
- 评分耗时统计

## 🛠️ 系统配置管理

### **环境变量配置（Render.com）**
在Render控制面板配置以下环境变量：

| 变量名 | 值 | 说明 |
|--------|-----|------|
| `GITHUB_TOKEN` | `github_pat_...` | GitHub个人访问令牌 |
| `GITHUB_OWNER` | `Gloria928` | GitHub用户名 |
| `GITHUB_REPO` | `ai-scenario-submissions` | GitHub仓库名 |
| `FRONTEND_URL` | `https://gitee.com` | 前端页面地址 |
| `NODE_ENV` | `production` | 生产环境 |
| `PORT` | `10000` | 服务端口 |

### **测试用例管理**
测试用例文件：`tests/test_cases.json`

**可以修改**：
- 添加新的作业题目
- 修改测试用例
- 调整评分规则
- 更新反馈模板

**修改后需要**：
1. 提交到GitHub仓库
2. Render会自动重新部署
3. 新测试用例立即生效

## 🎯 作业管理流程

### **1. 发布新作业**
1. 在Gitee教材中添加作业说明
2. 在 `tests/test_cases.json` 中添加测试用例
3. 提交到GitHub仓库
4. 通知学生开始提交

### **2. 监控作业提交**
**实时监控**：
- 系统统计页面：`/api/stats`
- GitHub Issues页面
- 自动评分工作流

**重点关注**：
- 提交高峰期（作业截止前）
- 提交失败的学生
- 评分异常的情况

### **3. 评分结果管理**
**自动评分**：
- GitHub Actions自动运行
- 评分结果添加到Issue评论
- 学生可以通过提交ID查询

**手动干预**：
- 对特殊情况进行手动评分
- 添加个性化评语
- 调整评分标准

### **4. 成绩统计分析**
**统计维度**：
- 班级整体成绩分布
- 各作业完成情况
- 学生进步趋势
- 常见错误分析

**导出数据**：
- 从GitHub Issues导出
- 从系统统计接口导出
- 手动整理分析

## 🔧 故障排除

### **常见问题及解决方案**

#### **1. 学生无法提交作业**
- **检查**：系统健康状态 `/health`
- **检查**：GitHub Token是否有效
- **检查**：网络连接是否正常

#### **2. 自动评分不运行**
- **检查**：GitHub Actions是否启用
- **检查**：Issue标签是否正确
- **检查**：测试用例文件格式

#### **3. 系统响应缓慢**
- **原因**：Render免费服务休眠
- **解决**：多次访问唤醒服务
- **建议**：高峰期提前访问

#### **4. 数据统计异常**
- **检查**：系统统计接口 `/api/stats`
- **检查**：GitHub Issues数据
- **检查**：环境变量配置

### **紧急处理流程**
1. **检查系统状态**：访问健康检查
2. **查看错误日志**：Render控制面板Logs
3. **重启服务**：Render手动重新部署
4. **联系技术支持**：顾三（当前助手）
